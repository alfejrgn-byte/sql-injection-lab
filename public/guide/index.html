Theory Guide

SQL Injection 이론 가이드

실습에 들어가기 전에 알아야 할 핵심 개념, SQLite 특성, 방어 방법을 빠르게 정리한 페이지입니다.

대시보드로 실습 시작

1. SQL Injection이란?

사용자 입력이 SQL 문장에 직접 결합될 때, 원래 의도하지 않은 쿼리 구조가 실행되는 취약점입니다.

애플리케이션이 문자열을 이어 붙여 쿼리를 만들면, 입력값이 데이터가 아니라 SQL 문법 일부로 해석될 수 있습니다.

2. SQL 기초 문법 복습

SELECT 는 조회, WHERE 는 조건, UNION 은 결과 합치기, ORDER BY 는 정렬입니다.

SELECT id, name
FROM products
WHERE category = 'electronics'
ORDER BY id DESC;

3. 인젝션 유형 분류

  • In-band: 응답 본문에 결과가 바로 보이는 방식
  • Blind: 참/거짓, 시간 차이처럼 간접 신호로 추론하는 방식
  • Out-of-band: DNS, HTTP 등 외부 채널을 통해 유출하는 방식

4. SQLite 특이사항

SQLite는 파일 기반 DB라 설치가 간단하고, 메타데이터는 sqlite_master 테이블에서 확인할 수 있습니다.

DBMS마다 함수 이름과 문자열 처리 방식이 다르므로, 페이로드는 사용 중인 엔진에 맞춰 조정해야 합니다.

5. 방어 기법

  • Prepared Statement 사용
  • 파라미터 바인딩으로 입력과 쿼리 구조 분리
  • ORM 사용 시에도 raw query 남용 금지
  • 에러 메시지 최소화와 로깅 분리

6. 윤리적 해킹

허가받지 않은 시스템에 대한 공격 시도는 불법입니다. 교육, 연구, 모의훈련처럼 명확히 승인된 범위에서만 테스트해야 합니다.

재현 환경, 책임 범위, 보고 절차를 사전에 정의하는 것이 기본입니다.
